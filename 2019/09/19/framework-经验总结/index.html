framework-经验总结

发表于 2019-09-19 | 分类于 Programming | 评论数：

SDK 简介

SDK 其实就是把一些通用代码的二进制、资源文件、和头文件打包起来使用，这样做主要有两个好处：

  1. 方便复用
  2. 隐藏实现细节

一般打包 SDK 有两种形式：

  1. xxx.framework
  2. xxx.a + xxx.h + xxx.bundle

接下来我们主要介绍 framework 封装方式。

打包 framework 的方式

有两种方式可以打包 framework ：

  1. Xcode 直接 build
  2. CocoaPods

Xcode 直接 build

主要就是通过 Xcode 创建 framework 后分别编译出模拟器版本SDK和真机版本SDK，然后合并两个SDK，得到最终的通用SDK

1️⃣.创建framework

  1. 打开xcode,新建iOS->Framework & Library->Cocoa Touch Framework

  2. 在PROJECT->Deployment Target->iOS Deployment Target选择你需要支持的最低系统。同样的操作在TARGETS中，Deployment Info->Deployment Target

  3. 在Build Settings->Linking->Mach-O Type->Static Library

  4. 这里要注意，在编译时，不要将图片文件，xib,媒体文件放在工程里面，否则编译后framework中会出现大量的零散图片文件在里面。这时需要将图片等资源放在.bundle文件中，在第三步介绍

  5. 这样打包的framework依然有问题，如果你用了Category，别人在用你的framework时会发生崩溃。这时别人在引用时需要在工程中other linker flags中添加-objC如果依然有问题，再添加-all_load

  6. 暴露公共的头文件以供使用者读取。在TARGETS->Build Phases->Headers里面，有三种类别。Public(公共的)，这里存放公开的头文件。Private(私有的)这里存放私有的Header，虽然说是私有的，但是还是公开的头文件，编译之后仍然可以framework包里面找到。所以有些Header是不想给别人看到的，必须要放在Project中

  7. 打包。Edit Scheme->Build Configuration->选为 Release 然后 Build 即可

注意：自己的SDK最好在每个类名前加个前缀，以防止下面未来可能发生的重复引入错误

附加：这样做出来的framework只能在模拟器或者真机上用，要想都可以用，请看下步

3️⃣.生成bundle文件
  1. 新建OS X->Framework & Library->Bundle
  2. 在TARGTS->Build Settings->Deployment->iOS Deployment Target->选择自己需要支持的最低系统
  3. build后会生成一个bundle包，但在包中的图片由以前的png格式全部变成tiff格式。为了防止这种格式转变。需要在Build Settings->Architectures->Base SDK->选择iOS的SDK要支持的版本。这时TARGETS中Build Setting->User-Defined中会出现一个新的Key：COMBINE_HIDPI_DEBUG_INFO,把它设置为NO
  4. 关于怎么取资源 提供LHBundleTool类获得

framework是里面代码的实现，xib、storyboard文件放在bundle项目工程里面。也就是两个项目，但是呢，只要storyboard上的类名和你要拖入的类名一致，还是可以照样拖线的。

5️⃣.第三方冲突的问题

有些必要的框架可能你SDK里面使用到了，项目工程里面也用到了，例如常用的AF,SD,MB等等，会有冲突。暂时自己的解决方案，在编写SDK引入框架的时候

以上：基本流程讲完了，重点是资源管理这一块，图片、xib、音视频等用bundle管理后，framework内访问资源的方式都要改成从具体的bundle访问，而不再是 mainBundle

CocoaPods 创建 framework

官方文档

市面上能搜到的大部分和打包framework相关的博客都是以上那种通过 Xcode 编译出来，先说上面那种方式的不足之处：

  1. 第三方依赖不好管理
  2. 不方便测试，不能直接断点到代码行
  3. 资源管理不方便

接下来我们开始一步步利用 CocoaPods 创建一个 framework

1️⃣.利用命令行创建模板项目

pod lib create MyLibrary

2️⃣.podspec

.podspec 文件描述了一个 pod 库的版本。它详细说明了这个 pod 库中源码应该从哪里取出、应用怎样的构建设置以及其他基本的信息，比如名称、版本、描述等。
podspec 文件的内容如下，并做了简要注释，不过还是强烈建议阅读下官方文档:

3️⃣.Development Pods

Development Pods 是符号链接文件，所以对它的编辑也会改变原始文件。这样就能在xcode内开发你的pod库。当你向 Pod/Classes 或 Pod/Assets 添加新的/已经存在的文件，或者更新你的 .podspec 时，需要再次运行 pod install 或者 pod update。

配置完 podspec 之后，需要对它进行验证，没有错误和警告就能通过验证了。

4️⃣.打包framework

安装 Cocoapods-package:
sudo gem install cocoapods-packager

打包命令:
pod package AC_OEM_iPhone.podspec --force --no-mangle

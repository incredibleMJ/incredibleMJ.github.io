Xcode黄色文件夹和蓝色文件夹区别，以及资源文件存储相关的一些思考

发表于 2017-12-06 | 分类于 Programming | 评论数：

文件夹区别

  1. 黄色文件夹是以 create groups 生成的，蓝色文件夹是以 create folder references 生成的

  2. group 下文件参与编译, reference 下文件不编译

  3. Xcode9 之前 group 文件结构与磁盘上是不一致的，新建 group 并不会在磁盘上新建文件夹， Xcode9 之后项目里的文件结构会和磁盘上保持一致，新建 group 也会在磁盘上新建文件夹，但保存资源文件时，运行时所有的资源文件都会被拷贝到 Bundle 里，也就是 .APP 文件夹下，此时在 group 下的资源文件直接会在 Bundle 根目录 , 而 reference 下文件在拷贝到 Bundle 时，同时会新建文件夹

怎么组织资源文件

  1. 代码放 group (黄色文件夹) 里，资源文件放 reference (蓝色文件夹) 里
  2. 对于资源文件 包含图片和其他
    • 图片可以放在 Assets.xcassets 里 或其他文件夹(包括自定义 Bundle )下
    • 其他资源文件不放 Assets.xcassets 里
    • 图片尽量放 Assets.xcassets 下，除非图片内存占用过高或某些使用频率低却大体积的图片文件放另外的文件夹下

图片获取的代码几点注意事项

  • 放在 Assets.xcassets 下的文件，用 imageNamed: 方法获取图片时,系统会在内存缓存，下次直接用，可能会造成内存占用过高
  • 放在文件夹下的图片，用 initWithContentsOfFile: 获取时，图片名要加上 @2x , @3x 这些信息，需要自己判断，而 assets.xcassets 下的图片直接写图片名就好了，系统会自动适配合适的图片
  • 放在 Assets.xcassets 以外的图片，用imageNamed:获取图片时，图片要在 Bundle 根目录下，图片名后缀要加上 (.jpg , .png 可以不用加)
